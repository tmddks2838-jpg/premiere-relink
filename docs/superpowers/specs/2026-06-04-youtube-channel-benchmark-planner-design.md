# 유튜브 채널 벤치마킹·역순 기획 툴 — 설계 문서

- **작성일**: 2026-06-04
- **작성자**: 김승안 + Codex
- **상태**: 설계 확정 (사용자 리뷰 대기)
- **작업명**: YouTube Benchmark Planner

## 1. 배경 & 문제 정의

유튜브 채널을 벤치마킹할 때 지금의 수작업은 시간이 오래 걸리고, 분석 결과가 사람의 감에 쉽게 흔들린다. 특히 "이 채널은 왜 잘 되는가"를 단순 조회수 목록이 아니라 실제 기획에 쓸 수 있는 형태로 바꾸는 과정이 어렵다.

이 툴은 사용자가 벤치마킹할 유튜브 채널 URL을 입력하면 최근 50개 영상 데이터를 수집하고, 성과가 좋은 영상의 제목·썸네일·초반 후킹 패턴을 분석한 뒤, 사용자의 채널 상황에 맞는 영상 기획안 5개를 생성한다.

핵심 차별점은 일반적인 "영상 아이디어 생성기"가 아니라 **제목 → 썸네일 → 첫 5초 → 첫 30초** 순서로 역산하는 기획 툴이라는 점이다. 사용자가 올린 후킹 이론 문서는 기본 내장 프레임워크로 압축해 MVP에 반영한다.

## 2. 프로젝트 범위

### 목표 (MVP)

- 웹앱 형태로 제공한다.
- 사용자가 YouTube API 키와 벤치마킹 채널 URL을 입력한다.
- 최근 50개 영상의 공개 메타데이터를 YouTube Data API로 수집한다.
- 가능한 경우 공개 자막/스크립트를 보강 분석한다.
- 채널 성과, 제목 패턴, 썸네일 패턴, 초반 후킹 구조를 분석한다.
- 사용자의 내 채널 정보는 선택 입력으로 받는다.
- 역순 기획안 5개를 생성한다.
- 결과를 화면에서 확인하고 Markdown으로 내보낼 수 있다.

### 비목표 (MVP에서 안 함)

- 여러 채널 동시 비교 분석.
- 로그인, 팀 기능, 클라우드 DB 저장.
- 사용자의 커스텀 후킹 파일 업로드.
- 썸네일 이미지 자체를 생성하거나 편집.
- YouTube Analytics 비공개 지표 분석.
- 영상 전체 내용의 완전한 의미 분석.

### 미래 확장

- v2: 2~5개 채널 비교 분석.
- v2: 커스텀 후킹/브랜드 톤 파일 업로드.
- v2: 분석 히스토리 DB 저장 및 재분석.
- v2: PDF/CSV 내보내기.
- v3: 썸네일 이미지 분석 또는 썸네일 시안 생성.
- v3: 내 채널 YouTube Analytics 연동.

## 3. 프로젝트 위치와 분리 원칙

현재 작업공간은 프리미어 미디어 재연결 툴이므로, 이 기능은 같은 코드베이스에 섞지 않는다. 구현은 별도 프로젝트로 시작한다.

권장 새 프로젝트 경로:

```text
/Users/gimseung-an/youtube-benchmark-planner
```

현재 저장소에는 이 설계 문서만 보관한다. 실제 구현 계획 수립 후 새 프로젝트 디렉토리를 만든다.

## 4. 제품 흐름

MVP의 사용자 흐름은 한 화면에서 단계적으로 이어진다.

```text
1. YouTube API 키 설정
2. 벤치마킹할 유튜브 채널 URL 입력
3. 선택 입력: 내 채널 정보 / 타깃 / 톤 / 피하고 싶은 스타일
4. 최근 50개 영상 데이터 수집
5. 채널 성과 분석
6. 제목·썸네일·후킹 패턴 분석
7. 기본 내장 후킹 프레임워크로 해석
8. 역순 기획안 5개 생성
9. 결과 저장 / Markdown 내보내기
```

결과 화면은 네 영역으로 구성한다.

```text
채널 요약
최근 50개 영상 성과표
성공 패턴 분석
역순 영상 기획안 5개
```

## 5. 사용자 입력

### 필수 입력

- YouTube API 키
- 벤치마킹 채널 URL

지원할 URL 형태:

```text
https://www.youtube.com/@handle
https://www.youtube.com/channel/UC...
https://www.youtube.com/c/name
https://www.youtube.com/user/name
```

MVP는 `@handle`과 `channel/UC...` 형태를 우선 안정 지원한다. `c/name`, `user/name`은 변환 가능한 경우 처리하고, 실패 시 명확한 오류 메시지를 보여준다.

### 선택 입력

사용자가 내 채널 정보를 입력하면 기획안을 더 맞춤화한다. 입력하지 않아도 일반 벤치마킹 분석과 기획안 생성은 진행된다.

- 내 채널 주제
- 타깃 시청자
- 원하는 영상 톤: 정보형, 재미형, 고급형, 자극형 등
- 피하고 싶은 스타일
- 선호 영상 길이
- 현재 고민: 조회수, 구독 전환, 후킹, 주제 고갈 등

## 6. 데이터 수집 설계

### YouTube Data API 흐름

```text
채널 URL
→ handle/channelId 추출
→ channels.list
→ 업로드 playlist ID 확보
→ playlistItems.list로 최근 50개 영상 ID 확보
→ videos.list로 통계/메타데이터 수집
→ 가능한 경우 자막/스크립트 보강
→ 분석 모델 생성
```

MVP의 기본 데이터는 YouTube Data API의 공개 메타데이터다. 공식 API 기반으로 시작해 수집 안정성과 재현성을 우선한다.

### 채널 데이터

- 채널명
- 설명
- 구독자 수
- 총 조회수
- 총 영상 수
- 업로드 플레이리스트 ID

### 영상 데이터

- 영상 ID
- 제목
- 설명
- 썸네일 URL
- 업로드일
- 길이
- 조회수
- 좋아요 수
- 댓글 수
- 태그
- 카테고리

### 자막/스크립트 처리

YouTube Data API의 공식 captions 다운로드는 권한과 쿼터 제약이 크므로, 경쟁 채널 전체 분석의 필수 경로로 두지 않는다. MVP는 항상 메타데이터 기반 분석을 수행하고, 가능한 영상에 대해서만 공개 자막/스크립트 수집 모듈로 첫 30초 분석을 보강한다.

자막 수집 실패는 전체 실패가 아니다. 결과에는 다음 상태를 표시한다.

```text
자막 분석 가능
자막 없음
자막 접근 실패
메타데이터 기반 추정만 사용
```

## 7. 분석 엔진

### 성과 분석

최근 50개 영상 기준으로 다음 지표를 계산한다.

- 평균 조회수
- 중앙값 조회수
- 조회수 상위 영상
- 채널 평균 대비 성과 배수
- 업로드 요일/시간 패턴
- 영상 길이별 성과
- 좋아요율
- 댓글율

성과 등급은 단순 조회수 순위만 쓰지 않는다. 최근 50개 평균과 중앙값을 함께 보고, 특정 영상이 채널 평균 대비 몇 배 성과를 냈는지 계산한다.

### 패턴 분석

- 제목 반복 구조
- 자주 쓰는 키워드
- 숫자, 질문, 부정, 비교, 폭로형 표현
- 설명/태그 기반 주제 클러스터
- 썸네일 문구 추정
- 검색 주도형 vs 발견 주도형 콘텐츠 추정

썸네일은 MVP에서 이미지 자체를 깊게 분석하지 않는다. 제목, 썸네일 URL, 영상 메타데이터, 모델 추론을 바탕으로 문구·구도 패턴을 추정한다. 이미지 분석은 v3 확장으로 둔다.

### 후킹 분석

후킹 분석은 두 층으로 구성한다.

```text
기본: 제목/설명/태그/성과/썸네일 메타 기반 후킹 유형 추론
보강: 자막이 있는 경우 첫 30초 실제 전개 분석
```

분석 결과는 영상별로 다음 태그를 가질 수 있다.

- 손실 회피
- 호기심 갭
- 자이가르닉 효과
- 반항 효과
- 통념 부정
- 인사이더 폭로
- 구체적 숫자
- 사례 기반
- 검색형
- 발견형

## 8. 내장 후킹 프레임워크

사용자가 첨부한 후킹 이론 문서는 원본 레퍼런스로 보관하고, MVP에는 압축된 기본 프레임워크를 내장한다. 원문을 매번 AI 입력에 통째로 넣지 않고, 실전 분석과 생성에 필요한 규칙표로 변환한다.

### 클릭 후킹

- 손실 회피
- 호기심 갭
- 반항 효과
- 통념 부정
- 구체적 숫자
- 인사이더 폭로

### 유지 후킹

- 자이가르닉 효과
- PAS
- AIDA
- BAB
- Hook · Story · Offer
- Big Idea
- 0-3-15-30-90 룰

### 한국 시장 보정

- 솔직함
- 억울함
- 또래감
- 근거 의심 대응
- 가성비 프레임
- 과한 자기자랑 회피
- 직설 명령형 CTA 완화

### 소재 선택

- 고통점 채굴
- Trend × Evergreen 교차
- 구체성 테스트
- 검색 주도형 vs 발견 주도형
- 변화 콘텐츠

### 시간 구조

기획안은 다음 시간 구조를 기본으로 점검한다.

```text
0~3초: 첫 인상 게이트
3~15초: 타깃/자격 검증 게이트
15~30초: 약속 구체화 게이트
30~90초: 본문 진입 게이트
엔딩 1분: 다음 영상 게이트
```

## 9. AI 기획 생성 구조

AI 생성은 자유 서술이 아니라 구조화된 JSON을 먼저 만들고, UI에서 사람이 읽기 쉬운 형태로 렌더링한다.

생성 흐름:

```text
1. 최근 50개 영상에서 성과 좋은 영상 추출
2. 제목/썸네일/후킹 패턴 요약
3. 내장 후킹 프레임워크로 패턴 해석
4. 사용자의 내 채널 정보가 있으면 보정
5. 역순 기획안 5개 생성
```

각 기획안은 다음 항목을 가진다.

- 아이디어 한 줄
- 제목 후보 3개
- 썸네일 후보 3개
- 첫 5초 후킹
- 첫 30초 전개
- 0-3-15-30-90 구조
- 본문 구성
- 사용한 후킹 메커니즘
- 벤치마킹 근거
- 내 채널 맞춤 변형
- 주의할 점

### 기획안 JSON 스키마 초안

```json
{
  "plans": [
    {
      "concept": "string",
      "titles": ["string", "string", "string"],

        {

        }
      ],
      "hook_5s": "string",
      "hook_30s": "string",
      "timeline": {
        "0_3": "string",
        "3_15": "string",
        "15_30": "string",
        "30_90": "string",
        "ending": "string"
      },
      "mechanisms": ["loss_aversion", "curiosity_gap"],
      "benchmark_basis": "string",
      "customization": "string",
      "risk_notes": "string"
    }
  ]
}
```

## 10. 웹앱 화면 설계

### 화면 구성

MVP는 한 화면 단계형 대시보드로 만든다.

```text
상단: API 키/설정
1단계: 벤치마킹 채널 URL 입력
2단계: 내 채널 정보 선택 입력
3단계: 분석 진행 상태
4단계: 결과 대시보드
5단계: 역순 기획안 5개
```

### 주요 상태

- 초기 상태: API 키와 채널 URL 입력
- 분석 중: 현재 단계 표시
- 부분 성공: 자막 일부 실패 등은 경고로 표시하고 분석 계속
- 실패: 채널 인식 실패, API 키 오류, 쿼터 초과 등 명확한 복구 안내
- 완료: 분석 대시보드와 기획안 표시

### 결과 내보내기

MVP는 Markdown 내보내기를 지원한다. Markdown에는 채널 요약, 성과 상위 영상, 후킹 패턴 요약, 기획안 5개가 포함된다.

## 11. 기술 스택

새 프로젝트의 기본 스택은 다음으로 잡는다.

```text
Frontend
- Next.js
- TypeScript
- 대시보드 UI
- 로컬 저장
- Markdown 내보내기

Backend
- Next.js API routes
- YouTube Data API 연동
- 자막 수집 모듈
- 분석/기획 생성 모듈

AI
- OpenAI API
- 구조화 JSON 출력
- 내장 후킹 프레임워크 프롬프트
```

Next.js API routes를 우선 선택한다. MVP는 프론트와 백엔드를 한 프로젝트에서 관리해 속도를 높이고, 필요해지면 FastAPI 또는 별도 백엔드로 분리한다.

## 12. 저장 설계

MVP는 로그인과 서버 DB 없이 브라우저 로컬 저장을 사용한다.

저장 대상:

- 마지막 API 키 입력 상태는 보안상 원칙적으로 저장하지 않는다.
- 최근 분석 결과는 사용자가 원할 때 로컬 저장한다.
- Markdown 내보내기로 결과를 파일화한다.

v2에서 계정 로그인과 분석 히스토리 DB 저장을 추가한다.

## 13. 에러 처리

반드시 처리할 오류:

- API 키 없음
- API 키 오류
- YouTube 쿼터 초과
- 채널 URL 인식 실패
- 채널을 찾을 수 없음
- 최근 영상이 0개이거나 비공개 중심인 채널
- 삭제/비공개 영상 포함
- 자막 없음
- 자막 수집 실패
- AI 생성 실패
- AI JSON 파싱 실패
- Markdown 내보내기 실패

오류는 가능한 한 전체 분석을 멈추지 않는 방향으로 처리한다. 예를 들어 자막 수집이 실패해도 메타데이터 기반 분석은 계속 진행한다.

### 통합 테스트

- 정상 채널 URL → 최근 50개 수집 → 분석 결과 생성
- 자막 없음 → 분석 계속
- API 키 오류 → 복구 가능한 오류 메시지 표시
- AI JSON 형식 오류 → 재시도 또는 사용자 안내

### 수동 QA

- 데스크톱 화면에서 결과 대시보드 가독성 확인
- 모바일 폭에서 입력/결과 카드가 깨지지 않는지 확인
- 긴 한국어 제목과 긴 기획안 텍스트가 레이아웃을 깨지 않는지 확인

## 15. 성공 기준

MVP 완료 기준:

1. 사용자가 채널 URL을 입력해 최근 50개 영상 데이터를 가져올 수 있다.
2. 채널 요약과 최근 50개 영상 성과표가 표시된다.
3. 성과 상위 영상의 제목·주제·후킹 패턴이 요약된다.
4. 자막이 없어도 분석이 완료된다.
5. 자막이 가능한 영상은 첫 30초 분석이 보강된다.
6. 역순 기획안 5개가 생성된다.
7. 각 기획안은 제목 후보 3개, 썸네일 후보 3개, 첫 5초, 첫 30초, 0-3-15-30-90 구조를 포함한다.
8. 내 채널 정보를 입력한 경우 기획안에 맞춤 변형이 반영된다.
9. 결과를 Markdown으로 내보낼 수 있다.
10. API 키 오류, 쿼터 초과, 채널 URL 실패, 자막 없음 오류가 사용자에게 명확히 표시된다.
